ABOUT

A personal project that helps parents find the right playground or park — not just a park, but one that's fenced, shaded, has a bathroom, and is actually toddler-friendly.

The problem

Map apps tell you a park exists. They don't tell you the things a parent of a little kid actually needs to know before loading everyone into the car: Is it fenced so a toddler can't bolt? Is there shade in July? A bathroom for the inevitable emergency? Somewhere to park? That information is buried in hundreds of reviews nobody has time to read.

How it works

The app reads the reviews for you. Here's the pipeline behind every search:

  1. Find the parks. The Google Places API returns nearby playgrounds and parks, along with their photos, ratings, hours, and recent Google reviews.
  2. Read the reviews with AI. Each park's reviews are sent to Google's Gemini 2.5 Flash-Lite model, which extracts structured answers to seven specific questions (fenced? shade? bathrooms? changing table? toddler-friendly? parking? tennis courts?) and writes a one-line summary for each.
  3. Cross-verify against the map. The same parks are looked up in OpenStreetMap, whose community has tagged real-world features — fences, restrooms, parking lots, splash pads, picnic tables, drinking fountains. Where the map confirms a feature, the app marks it verified.
  4. Layer in Google's own facts. Google publishes structured fields for some parks (has a restroom, good for children); those are treated as first-party truth and take precedence.

That's why each feature shows one of two badges: a ✓ from reviews (Gemini's read of what visitors said), or a ✓ verified (confirmed by map or Google data).

The six things that matter

🔒 Fenced 🌳 Shade 🚻 Bathrooms 👶 Toddler-friendly 🅿️ Parking 🎾 Tennis

With the details that matter tucked underneath — like whether the bathrooms have a 🧷 changing table, checked in both the reviews and the map data. Plus weather that suggests a move when it's useful ("Hot day — show shaded parks"), a one-tap 👶 Toddler-ready preset, an estimated drive time for nap-window planning, favorites and private notes, playdate-ready sharing that sends a park's known features (not just a link), and a one-tap way to resume your last search.

Built with

  • Frontend: HTML, CSS, and vanilla JavaScript written from scratch — no framework. Installable as a Progressive Web App with offline support.
  • Backend: Vercel serverless functions (Node) that keep the API keys server-side.
  • AI: Google Gemini 2.5 Flash-Lite for review analysis.
  • Data: Google Places API, OpenStreetMap (Overpass), Open-Meteo for weather, Nominatim for address search.
  • Maps: Leaflet with OpenStreetMap tiles.

Security & privacy

The app is built defensively: API keys never leave the server (all Google calls run through serverless functions), the page enforces a strict allowlist of what it's permitted to load and talk to (a Content-Security-Policy), and the one third-party library it uses is integrity-checked so a tampered copy would be rejected. Full details are in the project README.

There are no accounts and no tracking. Your favorites, notes, recent searches, and home location live only in your own browser — nothing is sent to a server or shared. Clearing your browser data clears them.

Made by AWW as a personal project — free to use, no ads. If a feature looks wrong for a specific park, it's usually because the underlying reviews or map data are sparse there. Spotted a bug or have an idea? Send feedback.
