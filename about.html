ABOUT

Built for busy parents who need to pick the right playground or park fast.

The problem

Finding a good playground should take ten seconds, not twenty browser tabs. Map apps tell you a park exists. They don't tell you what a parent actually needs to know before loading everyone into the car: Is it fenced so little ones stay safely inside? Is there shade in July? A bathroom for emergencies? Room to park? That information sits buried in hundreds of reviews nobody has time to read.

The solution

A free web app that answers those questions at a glance. Search any address, and every nearby park shows the features parents care about, read from real reviews by AI and cross-checked against map data. No account, no app store, no ads. Built to load fast, works on any phone, and installs like an app.

How it works

Behind every search:

  1. Find the parks. The Google Places API returns nearby playgrounds and parks with their photos, ratings, hours, and recent Google reviews.
  2. Read the reviews with AI. Each park's reviews go to Google's Gemini 2.5 Flash-Lite model, which answers seven specific questions (fenced? shade? bathrooms? changing table? toddler-friendly? parking? tennis courts?) and writes a one-line summary for each.
  3. Cross-verify against the map. The same parks are looked up in OpenStreetMap, where the community has tagged real-world features like fences, restrooms, parking lots, splash pads, and drinking fountains. When the map confirms a feature, it's marked verified.
  4. Layer in Google's own facts. Google publishes structured fields for some parks (has a restroom, good for children), treated as first-party truth that takes precedence.

So each feature is badged by source: a ✓ from reviews, or a ✓ verified by map or Google data.

You can also just ask. On any park's card, type a question ("Are there baby swings?", "Muddy after rain?") and Gemini answers from that park's own reviews, or says so when the reviews don't cover it.

🔒 Fenced 🌳 Shade 🚻 Bathrooms 👶 Toddler-friendly 🅿️ Parking 🎾 Tennis

Why I built it

As a busy parent trying to maximize quality family time and minimize screen time, I needed a purpose-built place for definitive information, not twenty minutes of scrolling reviews.

Curiosity drove the rest of the way. I wanted to explore the edge of what someone without a technical background could build relying on the utility I got as a user, and on my taste. The product, design, and security decisions were mine. Anthropic's Claude Code guided and coded the rest.

Built with

  • Frontend: HTML, CSS, and vanilla JavaScript, written from scratch with no framework. Installable as a Progressive Web App with offline support.
  • Backend: Vercel serverless functions (Node) that keep the API keys server-side.
  • AI: Google Gemini 2.5 Flash-Lite for review analysis.
  • Data: Google Places API, OpenStreetMap (Overpass), Open-Meteo for weather, Nominatim for address search.
  • Maps: Leaflet with OpenStreetMap tiles.

Security & privacy

API keys never leave the server (all Google calls run through serverless functions), a strict Content-Security-Policy limits what the page can load or talk to, and the one third-party library is integrity-checked so a tampered copy would be rejected. Full details are in the project README.

No accounts, no tracking. Your favorites, notes, recent searches, and home location live only in your browser; nothing is sent to a server or shared. Clearing your browser data clears them.
